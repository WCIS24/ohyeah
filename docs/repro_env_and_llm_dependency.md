# Repro Environment & LLM Dependency

Purpose:
- Clarify whether baseline generation depends on external LLMs and provide minimal reproducible commands.

How to use:
- Cite this document when describing baseline generation and reproducibility constraints.

## Baseline generation dependency
- The baseline generator uses a template function `placeholder_generate` that formats a snippet from retrieved chunks; no external LLM API call is invoked in this script.[EVIDENCE] scripts/run_baseline.py:50-51; scripts/run_baseline.py:128-141
- Therefore, the baseline QA path is fully local in the current repo; if a real LLM is required, the placeholder should be replaced in `scripts/run_baseline.py` or a new generator should be added explicitly (not implemented here).[EVIDENCE] scripts/run_baseline.py:50-51; scripts/run_baseline.py:128-141

## Minimal reproducible commands
- Retrieval-only (no QA generation):
  - `python scripts\eval_retrieval.py --config configs\eval_retrieval.yaml`
  - This computes Recall@K / MRR@K and writes metrics.json.[EVIDENCE] README.md:53-57; scripts/eval_retrieval.py:145-166
- Full baseline QA (template generator + QA eval):
  - `python scripts\run_baseline.py --config configs\run_baseline.yaml`
  - `python scripts\eval_qa.py --config configs\eval_qa.yaml --predictions outputs/<run_id>/predictions.jsonl --gold data/processed/dev.jsonl`
  - This yields predictions.jsonl and QA metrics.json.[EVIDENCE] README.md:59-69; scripts/run_baseline.py:128-142; scripts/eval_qa.py:120-129

## Reproducibility notes
- All scripts write outputs/<run_id>/config.yaml and logs.txt, and run_id is generated by UTC timestamp + UUID.[EVIDENCE] README.md:26-29; src/finder_rag/utils.py:11-17
